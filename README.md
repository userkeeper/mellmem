# MellMem v2 — Telegram Mini App

Библиотека мем-футажей с поиском, категориями, **загрузкой от пользователей**, модерацией и **рейтингом авторов**.

## Что нового в v2

- 📤 **Пользовательские загрузки** — юзеры кидают свои футажи прямо из миниапа
- 🔐 **Автоавторизация через Telegram** — юзернейм подтягивается из initData, без регистрации
- ⭐ **Рейтинг авторов** — кто загрузил, сколько скачиваний набрал
- 👤 **Профиль** — статы + список своих футажей со статусом модерации
- ✅ **Модерация** — всё от пользователей сначала идёт в очередь
- 🚫 **Бан** — можно забанить юзера одной кнопкой

## Стек

- **Фронт:** single-file HTML + Telegram Web App SDK (4 таба: каталог / загрузка / рейтинг / профиль)
- **Бэк:** Node.js + Express, ES modules
- **БД:** PostgreSQL с таблицами categories / users / footage / downloads_log
- **Хранилище:** Cloudflare R2 (без egress)
- **Превью:** ffmpeg на бэке
- **Auth:** HMAC-валидация Telegram initData (официальный метод)

## API

### Публичные эндпоинты

- `GET /api/footage` — все одобренные футажи (с данными аплоадера)
- `GET /api/categories` — список категорий с количеством
- `GET /api/leaderboard` — топ-50 авторов по скачиваниям
- `POST /api/download/:id` — инкрементит счётчик + засчитывает очко аплоадеру

### Пользовательские (требуют `x-tg-init-data`)

- `POST /api/user/upload` — загрузка футажа (multipart, уходит в `status=pending`)
  - Лимит: 30 МБ, макс 60 сек, макс 5 загрузок в час
- `GET /api/user/me` — свой профиль + список своих футажей

### Админские (требуют `x-admin-token`)

- `POST /api/admin/login` → `{ token }`
- `GET /api/admin/pending` — очередь модерации
- `POST /api/admin/moderate/:id` — `{ action: 'approve' | 'reject' }`
- `POST /api/admin/ban-user/:tg_id` — бан юзера
- `POST /api/admin/upload` — админская загрузка (сразу approved, до 100 МБ, без лимита длительности)
- `DELETE /api/admin/footage/:id`
- `POST/DELETE /api/admin/categories`
- `GET /api/admin/stats` — общие метрики

## Деплой на Railway

### 1. Репозиторий

Распакуй архив, запушь в GitHub (приватный).

### 2. Railway проект

- **New Project** → Deploy from GitHub repo
- Добавь плагин **PostgreSQL** — `DATABASE_URL` подхватится автоматом

### 3. Переменные окружения (Railway → Variables)

```
ADMIN_PASSWORD=сложный_пароль
JWT_SECRET=<openssl rand -hex 32>

R2_ACCOUNT_ID=...
R2_ACCESS_KEY_ID=...
R2_SECRET_ACCESS_KEY=...
R2_BUCKET_NAME=mellmem
R2_PUBLIC_URL=https://pub-xxx.r2.dev

BOT_TOKEN=<новый токен после revoke>
MINIAPP_URL=https://<your-app>.up.railway.app
```

### 4. Cloudflare R2

1. R2 → Create bucket `mellmem`
2. Settings → Public Access → Enable (получишь `pub-xxx.r2.dev`)
3. Manage R2 API Tokens → Create token с правами Read&Write на этот бакет

### 5. Два процесса

В Railway создай **второй сервис** в том же проекте, в нём Custom Start Command:
```
npm run bot
```

Первый сервис использует дефолтный `npm start` (веб-API).

### 6. Telegram BotFather

После revoke старого токена:

```
/newbot → сохрани новый токен в Railway Variables
/mybots → твой бот → Bot Settings → Configure Mini App → Enable
/mybots → твой бот → Bot Settings → Menu Button
   → URL: https://<your-app>.up.railway.app
   → Text: 📂 Открыть каталог
```

## Как это работает для пользователя

1. Юзер жмёт `/start` боту → кнопка "Открыть каталог"
2. Миниап открывается, Telegram автоматом передаёт `initData` с данными юзера (подписаны HMAC-SHA256, валидируются на бэке)
3. Юзер может искать, фильтровать, скачивать
4. Нажав **"Загрузить"** — выбирает файл, название, теги → отправляется в модерацию
5. В **"Профиле"** видит свои загрузки и статусы
6. В **"Рейтинге"** видит топ-50 авторов

## Как это работает для админа

1. Открываешь `/admin.html`, пароль из `ADMIN_PASSWORD`
2. Вверху статы: сколько футажей, сколько в очереди, скачиваний за 24ч, всего юзеров
3. **Модерация** — видишь присланные футажи с превью, тегами, автором → Одобрить / Отклонить / Забанить юзера
4. **Загрузить** — своя загрузка (сразу approved, до 100 МБ)
5. **Все футажи** — поиск, удаление
6. **Категории** — добавить / удалить

## Защита от абьюза

- Rate limit: 5 загрузок в час с одного Telegram ID
- Макс 30 МБ и 60 сек на пользовательский футаж
- Макс 10 тегов
- Все пользовательские загрузки идут в `pending` — в каталоге не появляются до одобрения
- Кнопка бана юзера одним нажатием из модерации

## Возможные расширения

- **Избранное** — юзер сохраняет футажи к себе
- **Комментарии** под футажом
- **Тренды** — сортировка по недавним скачиваниям
- **Soundboard-режим** — только аудио для Reels/Shorts
- **Реквест-форма** — юзер просит конкретный мем, админ добавляет
- **Монетизация**: премиум-теги, эксклюзивные футажи, подписка на нового-загрузки
- **Интеграция с твоим РКН.НЕТ** — cross-promo внутри миниапа
